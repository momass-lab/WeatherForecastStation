# ESP32 TinyML Weather Forecast Station

> **Pre-train a Deep Q-Network (DQN) agent on real historical weather data, export it to TensorFlow Lite, and run live 3-hour temperature forecasts directly on an ESP32 microcontroller.**

---

## 📐 Architecture

```
Visual Crossing API
      │
      ▼
weather_api.py ──────► weather_history.csv
      │
      ▼
train_weather_rl.py  (DQN · 1 000 episodes)
      │
      ├──► weather_rl_model.tflite   (quantized, ~4 KB)
      ├──► model_data.h              (C++ array, flash to ESP32)
      │
      └──► predict.py               (live PC inference / validation)

ESP32 + BME280
weather_station.ino + model_data.h
      └──► Serial: predicted temp every 60 s
```

## 🚀 Quick Start

### 1 — Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2 — Fetch real weather data

```bash
python weather_api.py
```

Outputs **`weather_history.csv`** with 356+ hourly records for Prague
(fields: `datetime`, `temp`, `humidity`, `pressure`, `target_temp_3h`).

> To change city, edit `LOCATION = "Prague"` in `weather_api.py`.

### 3 — Train the RL agent

```bash
python train_weather_rl.py
```

Runs **1 000 DQN episodes** over the real data.  
Produces:
- `weather_rl_model.tflite` — quantized model (~4 KB)
- `model_data.h` — C++ array ready for ESP32

Expected output every 100 episodes:
```
Episode: 0100/1000 | Total Reward (Negative Error): -14.73 | Epsilon: 0.61
...
Episode: 1000/1000 | Total Reward (Negative Error):  -8.21 | Epsilon: 0.05
```

### 4 — Run live PC inference

```bash
python predict.py
```

## 🤖 RL Design

| Component | Details |
|-----------|---------|
| **State** | `[norm_temp, norm_humidity, norm_pressure]` (3 floats) |
| **Action** | 41 discrete offset bins: −10 °C → +10 °C, step 0.5 °C |
| **Reward** | `−abs(predicted_temp − actual_temp_3h)` |
| **Algorithm** | Deep Q-Network (DQN) with replay buffer + target network |
| **Network** | `3 → Dense(16, relu) → Dense(16, relu) → 41` (linear) |
| **Parameters** | ~1 000 — fits comfortably on ESP32 SRAM |
| **Optimizer** | Adam, lr = 0.001, γ = 0.90, ε-decay = 0.995 |

### Normalization constants (keep identical in Python **and** C++)

| Feature | Mean | Std |
|---------|------|-----|
| Temperature | 15.0 °C | 35.0 |
| Humidity | 50.0 % | 50.0 |
| Pressure | 1013.0 hPa | 50.0 |

## 🔌 ESP32 Deployment

### Hardware

| Part | Notes |
|------|-------|
| ESP32 Dev Module | Any 38-pin or 30-pin variant |
| BME280 sensor | I²C — SDA = GPIO 21, SCL = GPIO 22 |

### Arduino Libraries (install via Library Manager)

```
TensorFlowLite_ESP32     – TensorFlow Authors
Adafruit BME280 Library  – Adafruit
Adafruit Unified Sensor  – Adafruit
```

### Flashing

1. Copy **`model_data.h`** (generated by training) into `esp32/`.
2. Open `esp32/weather_station.ino` in Arduino IDE.
3. Board: **ESP32 Dev Module** · Port: your COM port.
4. Upload → open **Serial Monitor at 115 200 baud**.

Every 60 seconds:
```
─────────────────────────────────────────────
  Current reading:
    Temp     :  7.2 °C
    Humidity : 68.0 %
    Pressure : 1017.5 hPa

  🌡  3-hour forecast:
    Predicted temp :  8.7 °C  (offset +1.5 °C)
    Agent Q-value  : -0.0804  (action #23)
─────────────────────────────────────────────
```

## ⚠️ Important — Normalization Parity

The C++ firmware **must use the same constants** as the Python training code. If you change the training normalization, update both files:

```cpp
// esp32/weather_station.ino
const float TEMP_MEAN = 15.0f;  const float TEMP_STD = 35.0f;
const float HUM_MEAN  = 50.0f;  const float HUM_STD  = 50.0f;
const float PRESS_MEAN= 1013.0f;const float PRESS_STD= 50.0f;
```
